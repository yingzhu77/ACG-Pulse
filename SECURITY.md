# 安全策略

## 支持范围

安全修复优先面向 `master` 分支的最新版本。历史提交和未经修改的个人部署不保证单独维护。

## 报告漏洞

请不要为安全漏洞创建公开 Issue，也不要在讨论区粘贴可利用细节、管理员凭据、Cookie、Token、数据库或完整日志。

推荐使用 GitHub 仓库的 **Security → Report a vulnerability** 私下报告。如果该入口尚未启用，请通过仓库所有者主页提供的联系方式进行私下联系，并注明“ACG Pulse 安全报告”。

报告建议包含：

- 受影响的版本或提交
- 问题类型与影响范围
- 最小复现步骤
- 必要的脱敏日志或截图
- 可行的修复建议（如有）

维护者确认后会尽快评估影响、准备修复并协调披露时间。修复发布前，请避免公开漏洞细节。

## 敏感信息

以下内容不得提交到 Git：

- `.env` 与生产环境配置
- `ADMIN_PASSWORD`、`ADMIN_JWT_SECRET`
- AI Provider API Key
- B站 Cookie、`SESSDATA`、`bili_jct`
- 数据库备份与生产日志

如果凭据已经泄露，请立即撤销或轮换，而不是只删除 Git 中的文件。B站 Cookie 泄露后应退出相关会话并重新登录；管理员 JWT Secret 轮换后，已有管理令牌会全部失效。

## 部署建议

- 对公网仅开放 80/443，通过反向代理访问应用。
- 应用端口绑定到 `127.0.0.1`，不要直接暴露数据库或 RSSHub。
- 使用强管理员密码和至少 32 字符的随机 JWT Secret。
- 部署前运行 `bash scripts/check-config.sh .env`。
- 定期检查容器健康、API 错误率、容量和备份可恢复性。
